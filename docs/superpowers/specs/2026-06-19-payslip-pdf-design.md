# Payslip PDF Download — Design Spec

**Date:** 2026-06-19  
**Status:** Approved

---

## Goal

Allow HR staff and employees to download a salary slip as a PDF file directly from the app. The PDF is generated server-side (FastAPI) and returned as a file download.

## Architecture

A new endpoint on the existing salary-slips router generates the PDF on demand from the already-stored `SalarySlip` row plus the associated `Employee` record. No new database tables are needed. The frontend fetches the binary response and triggers a browser file download.

**Tech Stack addition:** `fpdf2` (pure Python, no system dependencies)

---

## Backend

### New endpoint

```
GET /api/employees/{employee_id}/salary-slips/{year}/{month}/pdf
```

- **Auth:** same rule as the existing JSON `GET /{year}/{month}` — HR and above can access any employee; `employee` role can only access their own (`current_user.employee_id == employee_id`).
- **Response:** `StreamingResponse` with:
  - `Content-Type: application/pdf`
  - `Content-Disposition: attachment; filename="slip_{employee_id}_{year}_{month:02d}.pdf"`
- **Error cases:**
  - `404` if slip not found
  - `403` if employee role accessing another employee's slip

### New service function

**File:** `backend/app/services/pdf_service.py`

```python
def generate_salary_slip_pdf(slip: SalarySlip, employee: Employee) -> bytes:
    ...
```

Takes the ORM objects, returns raw PDF bytes. Uses `fpdf2` to build the document.

### PDF layout

Single A4 page. Two-column table (label | value). The currency code (e.g. INR, USD, GBP) is shown in the employee header block. Monetary values in the table are plain numbers — no currency symbol prefix, avoiding glyph-rendering issues with Helvetica for non-ASCII symbols (₹, £, €).

```

─────────────────────────────────────────────────────────
SALARY SLIP  ·  June 2025

Employee : Jane Doe                  ID : ACME-00042
Country  : IN                  Currency : INR

EARNINGS
┌──────────────────────────┬──────────────────┐
│ Gross Salary             │      95,000.00   │
└──────────────────────────┴──────────────────┘

DEDUCTIONS
┌──────────────────────────┬──────────────────┐
│ PF (Employee)            │       5,700.00   │
│ Income Tax               │       8,200.00   │
│ Other Deductions         │           0.00   │
└──────────────────────────┴──────────────────┘

NET TAKE-HOME                         81,100.00
─────────────────────────────────────────────────────────
Generated by: hr@acme.com  ·  Generated at: 19 Jun 2025 14:32 UTC
```

Fonts: fpdf2 built-in (Helvetica). No external font files needed.

## Frontend

### New API function

**File:** `frontend/src/lib/api.ts`

```ts
export async function downloadSlipPdf(
  employeeId: number,
  year: number,
  month: number,
): Promise<void>
```

Fetches the `/pdf` endpoint with `responseType: blob` (axios), creates an object URL, clicks a hidden `<a>` tag, then revokes the URL. Filename: `slip_{employeeId}_{year}_{month:02d}.pdf`.

### UI changes

**`frontend/src/pages/SalarySlips.tsx`** (employee self-view)  
Add a `Download PDF` button to each row in the slip history table. Button is `size="sm" variant="outline"`. On click, calls `downloadSlipPdf(user.employee_id, s.period_year, s.period_month)`. Shows a loading state while fetching.

**`frontend/src/pages/EmployeeDetail.tsx`** (HR view)  
Add a `Download PDF` button to each slip row in the employee detail salary-slips section. Same behaviour. Uses the viewed employee's ID.

## Error Handling

- If the slip does not exist → `404`, frontend shows a toast error.
- If the user lacks permission → `403`, frontend shows a toast error.
- Network/generation failure → frontend shows a toast error; no partial file is saved.

## Out of Scope

- Bulk PDF download (zip of multiple slips)
- Storing generated PDFs in the database or filesystem
- Email delivery of PDFs
- Custom branding / logo
